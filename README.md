# 不要关灯玩手机

管不住自己？交给程序。

自觉会失效，代码不会。

意志力会投降，程序不会。

你的眼睛，值得一个不会妥协的程序。

爱你眼睛的事，交给不偷懒的程序去做。

一款 Android 护眼应用，通过光线传感器实时监测环境亮度，在暗光环境下**强制锁定**手机，防止损害视力；当光线恢复后自动解锁。

> 技术栈：`Kotlin` · `Android SDK 34` · `minSdk 23 (6.0)` · `AccessibilityService` · `Light Sensor` · `Foreground Service`

## 功能特性

- 🔒 **强制锁定**：光线不足时全屏遮罩层，完全无法操作手机
- 🌙 **实时光线监测**：2000ms 智能采样间隔，持续监测环境亮度（lux），兼顾功耗
- 🎚 **阈值无极调节**：0-500 lux 滑块调节 + 手动精确输入
- ⏱ **防抖延迟触发**：可选开启，光线持续低于/高于阈值 3 秒才触发锁定/解锁，避免偶然遮挡导致频繁触发
- 📞 **电话场景智能处理**：来电、拨号器、通话中自动隐藏遮罩层，离开电话应用后自动恢复
- 🔦 **手电筒快捷开关**：遮罩层内一键开启手电筒
- 🚨 **紧急电话**：遮罩层内可拨打紧急电话
- 📊 **实时亮度显示**：通知栏同步显示当前环境亮度（仅展示，点击不跳转）
- 🌑 **息屏智能管理**：息屏自动暂停监测节省电量，亮屏解锁后自动恢复
- 🚀 **开机自启动**：支持开机自动启动保护（需手动开启）
- ⚙️ **稳定运行指引**：内置 5 步设置引导，确保后台稳定运行
- 👴 **老年友好设计**：大字体（标题 48sp，正文 28sp）、大按钮（最小高度 72dp）

### 三状态防抖状态机

开启防抖模式后，锁定/解锁通过三状态机控制，避免光线波动导致频繁切换：

```
         光线 < 阈值 持续 3 秒
  ┌──────────┐ ──────────────────────→ ┌──────────┐
  │   IDLE   │                         │  LOCKED  │
  │ (空闲态)  │ ←────────────────────── │ (锁定态)  │
  └──────────┘   冷却期 3 秒结束后      └──────────┘
       ↑                                  │
       │         光线 ≥ 阈值              │ 光线 ≥ 阈值
       │       ┌──────────┐               │
       └────── │ COOLDOWN │ ←─────────────┘
               │ (冷却态)  │   进入 3 秒冷却期
               └──────────┘
```

- **IDLE（空闲态）**：正常监听，光线持续低于阈值 3 秒后触发锁定
- **LOCKED（锁定态）**：遮罩层显示中，光线恢复到阈值以上立即解锁
- **COOLDOWN（冷却态）**：解锁后强制 3 秒冷却，忽略光线变化，防止反复触发

### 电话场景处理

通话相关场景优先级最高，不受锁定状态限制：

1. **来电响铃** → 自动移除遮罩层，让用户接听
2. **拨号器/通话界面** → 无障碍服务检测到电话应用前台时，隐藏遮罩层
3. **用户点击紧急电话按钮** → 设置 `isPhoneAppActive` 标记，移除遮罩层
4. **离开电话应用** → 自动恢复遮罩层（60 秒超时兜底）
5. **全程保持 `isLocked = true`** → 不改变锁定状态，仅控制遮罩层 UI 显示

## 使用说明

### 首次使用

1. 安装 APK 后打开应用
2. **授权引导页**：依次开启两项必要权限
   - **悬浮窗权限**：用于显示锁定遮罩层
   - **无障碍权限**：用于检测窗口变化，防止用户切换到其他应用
3. 两项权限均开启后，点击「进入应用」
4. **设置亮度阈值**：通过滑块调节（0-500 lux），或点击数值手动输入
5. **开启防抖模式**（可选）：开启后光线需持续低于/高于阈值 3 秒才触发
6. 点击「运行」开始保护
7. **稳定运行设置**（推荐）：点击「⚙️ 稳定运行设置指引」，按引导完成 5 项设置

### 亮度阈值参考


| 环境场景     | 亮度范围      | 建议阈值    |
| ------------ | ------------- | ----------- |
| 夜晚室内关灯 | 0 ~ 10 lux    | 10（严格）  |
| 昏暗环境     | 10 ~ 50 lux   | 50（标准）  |
| 夜晚室内开灯 | 50 ~ 200 lux  | 200（宽松） |
| 明亮环境     | 300 ~ 500 lux | 不需要保护  |

### 锁定界面

当环境光线低于设定阈值时，手机将被锁定并显示全屏遮罩层：

- 遮罩层显示当前亮度和目标亮度，以及亮度进度条
- **🔦 手电筒按钮**（左下角）：一键开启/关闭手电筒
- **📞 紧急电话按钮**（右下角）：打开拨号盘拨打紧急电话
- 除手电筒和紧急电话外，所有触摸操作均被遮罩层拦截

### 稳定运行设置

为确保应用在后台持续稳定运行，建议按以下顺序设置：

1. **允许后台运行**：设置 → 应用 → 不要关灯玩手机 → 电池 → 允许后台活动
2. **忽略电池优化**：将本应用设置为「不优化」或「无限制」
3. **后台管理**：关闭自动管理，手动开启自启动、关联启动、后台运行
4. **应用后台锁定**：从最近任务中长按本应用卡片，点击锁图标锁定
5. **关闭省电模式**：省电模式可能会杀死后台服务

> 不同品牌手机设置路径可能略有差异，应用内已提供详细引导。

## 技术实现

### 遮罩层防护

遮罩层通过全屏悬浮窗覆盖屏幕，消费所有触摸事件，阻止用户继续使用手机：


| 操作      | 防护方式                         | 效果      |
| --------- | -------------------------------- | --------- |
| 点击/滑动 | 悬浮窗全屏覆盖，消费所有触摸事件 | ✅ 被拦截 |

### 省电策略

- **WakeLock**：`PARTIAL_WAKE_LOCK` 保持 CPU 唤醒，确保息屏后传感器数据正常投递
- **息屏暂停**：收到 `ACTION_SCREEN_OFF` 广播后立即暂停传感器监听，释放 WakeLock
- **亮屏恢复**：用户解锁（`ACTION_USER_PRESENT`）后恢复监测，并立即检查当前光线状态
- **安全网机制**：传感器回调中主动检测屏幕状态，防止 `SCREEN_OFF` 广播丢失
- **传感器采样优化**：自定义 2000ms 软件节流，环境光变化缓慢无需高频采样
- **通知节流**：仅在锁定状态变化或亮度整数值变化 ≥ 5 时才更新通知
- **通知简洁化**：通知栏仅展示亮度信息，无标题、无停止按钮、点击无跳转
- **View 缓存复用**：遮罩层 View 仅首次 inflate，后续从 WindowManager 移除/添加，不重复创建

### 日志系统

- `AppLog` 统一日志工具，支持分级控制（Debug / 状态机 / 传感器）
- `DebugConfig` 全局调试开关，正式发布时设为 `false` 可关闭所有调试日志
- Warning 和 Error 级别日志始终输出

| -------------------------------------- | ------------------------------------- | ---------------------------- |

| `BIND_ACCESSIBILITY_SERVICE`           | 检测窗口变化，防止用户切换到其他应用  | 引导页跳转设置               |
| `POST_NOTIFICATIONS`                   | 显示前台服务通知（Android 13+）       | 点击「运行」时自动请求       |
| `CAMERA`                               | 控制手电筒（闪光灯）                  | 点击「运行」时自动请求       |
| `READ_PHONE_STATE`                     | 监听通话状态（来电/去电）             | 点击「运行」时自动请求       |
| `FOREGROUND_SERVICE`                   | 保持后台光线监测服务运行              | 安装时自动授予               |
| `FOREGROUND_SERVICE_SPECIAL_USE`       | 前台服务类型声明（Android 14+）       | 安装时自动授予               |
| `WAKE_LOCK`                            | 保持 CPU 唤醒，确保传感器数据正常投递 | 安装时自动授予               |
| `CALL_PHONE`                           | 打开紧急拨号界面                      | 安装时自动授予               |
| `RECEIVE_BOOT_COMPLETED`               | 接收开机广播，支持自启动              | 安装时自动授予               |
| `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` | 请求忽略电池优化，保持后台稳定运行    | 设置指引页跳转               |

| 依赖                  | 版本  |

### 使用 Android Studio

1. 克隆项目并打开 `eyeguard` 目录
2. 修改 `local.properties`，设置你的 SDK 路径：
   ```
   sdk.dir=C:\\Users\\<用户名>\\AppData\\Local\\Android\\Sdk
   ```
3. 点击 **Build > Build Bundle(s) / APK(s) > Build APK(s)**
4. APK 将生成在 `app/build/outputs/apk/debug/` 目录

### 使用命令行

```bash
# 进入项目目录
cd eyeguard

# 构建 Debug APK
./gradlew assembleDebug

# 构建 Release APK
./gradlew assembleRelease

# 安装到设备
adb install app/build/outputs/apk/release/eyeguard_v1.1.1_release.apk
```

## 注意事项

1. **权限必须开启**：缺少悬浮窗或无障碍权限，应用无法正常工作
2. **电池优化**：部分厂商系统（小米、华为、OPPO、vivo 等）可能会杀死后台服务，务必按照「稳定运行设置指引」完成配置
3. **传感器差异**：不同设备光线传感器精度不同，可根据实际情况调整阈值
4. **正式发布**：构建发布版本前，请将 `DebugConfig` 中所有开关设为 `false`
